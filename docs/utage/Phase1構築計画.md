# BizOwner Phase 1 構築計画書（実装手順詳細）

> **目的**：BizOwnerファネルの「動く骨格」を最短で組み上げ、クライアントに見せて方向性を確定させる。
> **完成定義**：LINE登録 → アンケート → ウェビナー視聴 → 決済 → メンバーサイト招待 までが1人のテストユーザーで通しで動く状態。
> **想定期間**：実作業ベースで5〜7営業日（クライアント確認・素材待ち時間は別）。

---

## 📌 Phase 1 のスコープ確認

### やること（MVP）
- LINE：あいさつ1パターン／アンケート4-5問／リッチメニュー4マス／BO用ステップ配信5通
- ウェビナー：BO用1本（仮素材OK）／申込フォーム／リマインド2通
- 決済：BO月額11,000円フォーム＋学び家オプションチェック／法人作成10万円スポット
- メンバーサイト：構造とボタンだけ／中身は「準備中」プレースホルダ
- アフィリ：紹介URL自動発行／紹介人数表示のみ
- 個別面談：予約カレンダー1種／ZOOM連携／事前ヒアリング3問

### やらないこと（Phase 2以降）
- SARUDEMO・学び家のシナリオ
- ABテストの並走
- ウェビナー複数本化
- FC加盟決済・2期目切替
- 報酬自動計算ロジック
- 実コンテンツ動画の本格搭載

## 📅 実装手順（ステップ・バイ・ステップ）

### STEP 0：UTAGE初期設定（0.5日）

**目的**：UTAGEを使える状態にする

1. UTAGEログイン → 基本情報登録
   - 会社情報（社名・所在地・連絡先）
   - 特商法表記の登録
   - プライバシーポリシーURLの設定
2. 決済連携設定
   - Stripe または Univapay のAPIキー登録
   - テスト決済でフロー確認
3. メール送信設定（SPF/DKIM）
   - 独自ドメイン使用ならDNS設定
4. LINE公式アカウント連携
   - チャネルアクセストークン入力
   - Webhook URL設定
   - LINE側の応答モード「Bot」に変更
5. ZOOM連携
   - ZOOM APIキー登録 → 接続テスト

**完了基準**：管理画面ログイン・テスト決済成功・LINEテスト送信成功

---

### STEP 1：LINEシナリオ構築（1日）

**目的**：LP→LINE登録後、ユーザーが自動で教育される導線を作る

#### 1-1. タグ設計（最初に決める）
作るタグ一覧：
- `source_BO`（BizOwner LP経由）
- `source_SARU`（SARUDEMO LP経由・Phase 2用に枠だけ）
- `source_MANABIYA`（学び家 LP経由・Phase 2用に枠だけ）
- `credit_black` / `credit_white`（信用情報）
- `interest_BO` / `interest_SARU` / `interest_MANABIYA`（興味サービス）
- `seriousness_high` / `seriousness_mid` / `seriousness_low`（本気度）
- `webinar_watched_BO`（ウェビナー視聴済み）
- `paid_BO`（決済完了）

#### 1-2. あいさつメッセージ作成
登録直後に自動配信：
- ウェルカム挨拶（顧問の自己紹介・LP内容の補足）
- アンケート回答誘導（リッチメッセージ＋ボタン）
- リッチメニュー設置の案内

#### 1-3. 初回アンケートフォーム作成（UTAGEフォーム機能）
質問項目：
1. お名前
2. 年齢層（20代/30代/40代/50代以上）
3. 信用情報の状況（クレカ審査通る/通らない/分からない）
4. 副業・起業の経験（未経験/副業経験あり/起業経験あり）
5. 本気度（情報収集中/良い案件があれば始めたい/今すぐ始めたい）
6. 学び家（福利厚生コミュニティ）にも興味があるか（はい/いいえ/まだ分からない）

→ 回答に応じてタグ自動付与（UTAGEのフォーム→タグ連動機能）

#### 1-4. リッチメニュー作成（4マス）
| 左上 | 右上 |
|---|---|
| BizOwnerとは？（LP再表示） | ウェビナー予約 |
| **左下** | **右下** |
| 個別面談予約（任意） | よくある質問 |

→ Canva等で画像作成 → UTAGEからLINE側に配信

#### 1-5. BO用ステップ配信5通作成
| Day | テーマ | 内容 |
|---|---|---|
| Day 0（登録直後） | あいさつ＋アンケート | 上記1-2 |
| Day 1 | 共感＋問題提起 | 「副業に興味あるけど何していいか分からない」共感ストーリー |
| Day 2 | 解決策提示 | BizOwnerの3ステップ（個人案件→法人案件→FC化）紹介 |
| Day 3 | ウェビナー誘導 | 「詳しくはウェビナーで」予約フォームURL送付 |
| Day 5 | 学び家オプション訴求 | BO会員の福利厚生としての学び家紹介 |
| Day 7 | 決済CTA＋限定オファー | 月額11,000円・初月無料・解約自由 |

→ UTAGEシナリオ機能で「source_BO」タグ保有者のみ配信

**完了基準**：テストアカウントで登録→アンケート→Day7まで配信が自動で流れる

---

### STEP 2：ウェビナー構築（1日）

**目的**：BO用ウェビナーを録画動画ベースで自動配信できる状態にする

#### 2-1. ウェビナー動画の準備
- クライアントから提供 or 仮素材で進める
- 60〜90分想定／資料①〜⑤の内容を映像化
- UTAGEに動画アップロード（または外部動画URL登録）

#### 2-2. ウェビナー申込ページ作成
UTAGEのウェビナーテンプレート使用：
- タイトル：「BizOwner公式オンライン説明会」
- 日時選択（毎日複数枠／自動繰り返し設定）
- 申込フォーム（名前・メアド・LINE連携済みなら省略可）

#### 2-3. リマインド配信設定
| タイミング | 媒体 | 内容 |
|---|---|---|
| 申込直後 | メール＋LINE | 申込完了＋日時案内 |
| 開始1時間前 | LINE | リマインド＋視聴URL |
| 開始時刻 | LINE | 「今すぐ視聴できます」 |

#### 2-4. 視聴ページ作成
- 動画埋め込み
- 視聴ログ計測ON（離脱ポイント分析用）
- 動画下部に2つのCTA：
  - **[今すぐ申し込む]** → 決済フォームへ
  - **[個別に質問する]** → 面談予約フォームへ
- 視聴完了時に `webinar_watched_BO` タグ自動付与

**完了基準**：申込→リマインド→視聴→CTAクリックの一連が動く

---

### STEP 3：決済フォーム構築（1日）

**目的**：BO月額・法人作成・学び家オプションの3つの決済が動く

#### 3-1. BizOwner月額決済フォーム
- 商品設定：月額11,000円（税込）／初月無料
- 解約条件の表記
- **オプション欄**：☑ 学び家を追加する（+月額10,000円）
- フォーム項目：氏名・住所・電話・メアド・支払い情報
- 決済完了後の処理：
  - `paid_BO` タグ付与
  - 学び家チェック時は `paid_MANABIYA` タグ付与
  - メンバーサイトへ自動招待メール送信
  - LINE側にサンクスメッセージ自動配信

#### 3-2. 法人作成スポット決済フォーム
- 商品設定：100,000円スポット
- メンバーサイトの「STEP2解放」ボタンから呼び出し
- 決済完了後：`paid_corp_BO` タグ付与＋メンバーサイトのSTEP2エリア解放

#### 3-3. 特商法・利用規約の同意チェックボックス
全フォームに必須

**完了基準**：テストカードで3パターンの決済が成功し、タグが正しく付与される

---

### STEP 4：メンバーサイト構築（1日）

**目的**：会員専用ログインサイトを骨格だけ作る（中身は後で）

#### 4-1. UTAGEメンバーサイト機能で土台作成
- サイト名：BizOwner会員サイト
- ログイン方式：メアド＋パスワード（決済時に自動発行）
- デザインテンプレート選択

#### 4-2. ページ構造作成
| ページ | 表示条件 | 中身（Phase 1） |
|---|---|---|
| トップ（マイページ） | 全会員 | 会員ランク表示・お知らせ・紹介URL表示 |
| STEP1：個人案件 | `paid_BO` タグ保有 | 動画プレースホルダ5本（「準備中」表示） |
| STEP2：法人案件 | `paid_corp_BO` タグ保有 | 解放ボタンと「準備中」プレースホルダ |
| STEP3：FC加盟 | （Phase 1非公開） | プレースホルダのみ |
| 学び家エリア | `paid_MANABIYA` タグ保有 | 「コンテンツ準備中」プレースホルダ |
| 紹介プログラム | 全会員 | 紹介URL・人数表示 |

#### 4-3. アクセス制御設定
タグベースで表示エリアを制限（UTAGE標準機能）

**完了基準**：テスト会員でログイン→自分のランクに応じたエリアだけ見える

---

### STEP 5：アフィリエイト（紹介機能）構築（0.5日）

**目的**：紹介URLが自動発行され、紹介人数が見える状態にする

1. UTAGEのアフィリ機能ON
2. 紹介URL命名規則設定（例：`?ref=会員ID`）
3. メンバーサイトのマイページに以下を表示：
   - 自分の紹介URL（コピーボタン付き）
   - 紹介人数カウント
   - 紹介経由の決済件数
4. 報酬計算ロジックは Phase 1ではOFF（手動運用）

**完了基準**：会員Aが自分のURL経由でテスト登録Bを獲得→Aのマイページに紹介人数1と表示

---

### STEP 6：個別面談予約構築（0.5日）

**目的**：質問者・FC希望者が任意で予約できるカレンダーを作る

1. UTAGEカレンダー機能で予約枠設定
   - 平日10時〜18時／30分枠
   - 1日最大予約数制限
2. 予約フォーム
   - 氏名・連絡先・相談内容（自由記述3行）
3. ZOOM連携で会議URL自動発行
4. リマインド設定（前日／当日1時間前）
5. リッチメニュー・ウェビナー視聴ページからこのフォームへリンク

**完了基準**：テスト予約→ZOOM URL自動発行→リマインド到達

---

### STEP 7：通しテスト（0.5日）

**目的**：登録〜決済〜メンバーサイト招待まで1人のテストユーザーで通す

#### テストシナリオ
1. テスト用LINEから登録
2. アンケート回答（BizOwner興味・学び家も興味あり）
3. ステップ配信が順次届くか確認（時間短縮設定で）
4. ウェビナー予約→リマインド受信→視聴→CTAクリック
5. 決済フォームに「学び家オプション」チェック→テストカード決済
6. メンバーサイトに招待メールが届く→ログイン→BO/学び家エリアが見える
7. 紹介URLをコピー→別ブラウザで踏む→紹介人数が増えるか確認
8. 個別面談予約→ZOOM URL受信

#### チェックリスト
- [ ] タグが正しく付与される
- [ ] シナリオ分岐が意図通り動く
- [ ] 決済が完走する
- [ ] メンバーサイトのアクセス制御が効いている
- [ ] リマインドが指定時刻に届く
- [ ] 紹介URLが機能する

## ⚠️ 留意事項

- **仮素材で進める部分が多い**：動画・コピー・画像は本番素材待ちでもOK。骨格優先。
- **本番運用前に必ず特商法・利用規約・プライバシーポリシーをチェック**
- **テスト決済は本番カードで誤動作させないよう、Stripe/Univapayのテストモードを使用**
- **LINE配信は審査・配信制限がある**ため、本番リリース時にプレミアムID等の状況を再確認
- **Phase 1ではABテスト・複数シナリオを実装しない**：Phase 2で展開
